# Project: learngermanwithjonas – Deutschlehrer-Website (Jonas)
You are building a modern, professional landing page for learngermanwithjonas, the personal brand of Jonas, a German teacher.

## Goal
Create a clean, premium, conversion-focused single-page website that does NOT look AI-generated.
It should feel modern, trustworthy and human – a real teacher, not a faceless platform.
This is Jonas's own teaching site (private German lessons, incl. Pflegedeutsch), separate from Expatly,
but it reuses the same visual design language.

## Brand
- Brand name: learngermanwithjonas
- Person: Jonas, German teacher (private lessons, conversation, Pflegedeutsch)
- Audience: people living in Germany who want to actually speak German
- Tone: warm, direct, personal, practical, first person ("I help you speak")
- Avoid the word "expats" in headlines/hero – use "people who moved to Germany" or similar
- Avoid: generic AI gradients, boring cards, fake stock-photo feeling, weak copy

## Design System (inherited from Expatly – keep identical)
- Font: Rubik
- Background / creme: #FBF5F2
- Black: #1A1A1A
- Orange (accent / CTA): #FF881A
- Aesthetic: brutalist cartoon – thick black outlines, hard offset shadows (no soft blur shadows)
- Rounded corners consistent with Expatly components
- Reuse brand assets from /brand_assets whenever available (icons, illustrations, shapes)

## Design Rules
- Always design the polished frontend before writing code.
- Strong spacing, clean typography, consistent visual hierarchy.
- Hero must be immediately clear: who Jonas is + what the visitor gets.
- Real sections: hero, benefits, how it works, about Jonas / social proof,
  lessons/offer area, final CTA (booking or contact).
- Fully responsive for mobile, tablet, desktop.

## Development Rules
- First build locally.
- Use screenshots to visually inspect the result.
- Iterate before saying the design is done.
- Keep files organized.
- Do not use fake buttons without telling me where links need to be added
  (e.g. booking link, contact form, calendar).
- Do not add secrets, API keys, passwords or private data to GitHub.

## Local Preview
When changes are made, run the local dev server and show me the localhost URL.

## Deployment Rule
Only commit and push changes when I explicitly approve the local version.
Never push to GitHub or deploy unless I explicitly say: "push to GitHub".
